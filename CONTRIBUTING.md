# Bijdragen (CONTRIBUTING)

## Voordat je begint

- Controleer open issues om te zien of er al een discussie of bugreport is over wat je wilt wijzigen.
- Als je een nieuw groot feature wil toevoegen: open eerst een issue om de aanpak te bespreken.

## Kleine wijziging (bugfix, documentatie)

1. Fork het repository en maak een feature-branch met een duidelijke naam, bijvoorbeeld `fix/typo-readme` of `feat/widget-cijfers`.
2. Maak je wijziging in die branch.
3. Zorg dat je commit-berichten informatief zijn.
4. Push je branch en maak een pull request naar `main` in dit repository.

## Pull request checklist

- Beschrijf kort wat je wijziging doet en waarom.
- Voeg, indien van toepassing, een screenshot of korte instructie toe om de wijziging te testen.
- Houd de scope van de PR klein en gefocust.

## Code stijl

- Houd je aan bestaande stijl in het project (ES modules, kleine, leesbare functies).
- Gebruik duidelijke variabelen- en functienamen.

## Tests

Dit project bevat (voorlopig) geen automatische test-suite. Voor substantiele wijzigingen:

- Beschrijf handmatige teststappen in je PR.
- Voeg waar mogelijk kleine voorbeelden of een terugvalconfiguratie toe.

## Licentie en rechten

Door bij te dragen ga je akkoord met de licentie van dit project (zie `LICENSE`).
